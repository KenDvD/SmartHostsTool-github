# 🚀 智能Host测速工具

一个强大的智能域名IP测速与Hosts文件管理工具，帮助您轻松优化网络访问速度！

**注意：** 本项目的远程hosts文件由 [GitHub520](https://github.com/521xueweihan/GitHub520) 提供。

## ✨ 功能特点

### 🎯 核心功能
- **智能解析IP**：自动获取目标域名的多个可用IP地址
- **延迟测试**：对获取到的IP进行精准延迟测试并智能排序
- **一键写入Hosts**：将延迟最低的IP自动写入系统Hosts文件
- **DNS刷新**：一键清除DNS缓存，确保修改立即生效

### 💎 特色功能
- **GitHub专属优化**：支持刷新GitHub远程Hosts列表，获取更多可用IP
- **自定义预设管理**：轻松添加、删除和管理个性化域名列表
- **现代化GUI界面**：采用ttkbootstrap打造的精美可视化界面
- **个性化头像**：关于界面展示专属头像，提升用户体验

## 📖 使用方法

### 1️⃣ 运行准备
- **以管理员身份运行**：由于需要修改系统Hosts文件，请确保以管理员权限启动程序

### 2️⃣ 选择目标域名
- 点击「自定义网站预设」，选择需要测速的域名
- 支持多选模式，可同时测试多个域名

### 3️⃣ 获取IP地址
- 对于GitHub域名，可选择「刷新远程Hosts」获取更多IP资源
- 点击「智能解析IP」按钮，自动获取目标域名的可用IP

### 4️⃣ 开始测速
- 点击「开始测速」，程序将自动测试所有IP的延迟
- 测试结果按延迟时间自动排序，清晰直观

### 5️⃣ 写入Hosts
- 手动选择延迟较低的IP，点击「写入Hosts」
- 或使用「一键写入最优IP」功能，自动选择延迟最低的IP

### 6️⃣ 刷新DNS
- 点击「刷新DNS」按钮，使Hosts文件修改立即生效

## ❓ 常见问题

### Q: 为什么程序需要管理员权限？
A: 因为系统Hosts文件属于系统级文件，修改此类文件需要管理员权限才能执行。

### Q: 为什么测试结果显示某些IP延迟很高？
A: 测速结果受多种因素影响，包括网络环境、路由状况、服务器负载等，建议多次测试取平均值以获得更准确的结果。

### Q: 如何添加自定义域名？
A: 点击「添加预设」按钮，在弹出的对话框中输入您需要的域名（如：example.com），然后点击确定即可完成添加。

### Q: 为什么GitHub域名有特殊处理？
A: GitHub在全球分布有多个服务器节点，通过刷新远程Hosts列表可以获取更多可用的IP地址，从而提高连接成功率和访问速度。
